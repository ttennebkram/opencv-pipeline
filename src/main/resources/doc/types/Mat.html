Mat

The Mat class represents an n-dimensional dense numerical array. It is the primary data structure used by OpenCV to store and manipulate images.

Overview

A Mat object consists of:

  • Header - Contains metadata like size, type, and reference count
  • Data pointer - Points to the actual pixel data in memory

Multiple Mat objects can share the same data (shallow copy). Use clone() to create a deep copy with its own data.

Common Properties

PropertyDescription
rows()Number of rows (height)
cols()Number of columns (width)
width()Same as cols()
height()Same as rows()
channels()Number of channels (1 for grayscale, 3 for BGR, 4 for BGRA)
type()Type identifier (e.g., CV_8UC3)
depth()Bit depth (e.g., CV_8U, CV_32F)
empty()Returns true if Mat has no data

Common Types

TypeDescription
CV_8UC18-bit unsigned, 1 channel (grayscale)
CV_8UC38-bit unsigned, 3 channels (BGR color)
CV_8UC48-bit unsigned, 4 channels (BGRA with alpha)
CV_32FC132-bit float, 1 channel
CV_32FC332-bit float, 3 channels

Common Methods

MethodDescription
clone()Creates a deep copy of the Mat
copyTo(dst)Copies data to another Mat
convertTo(dst, type)Converts to different type/depth
submat(roi)Extracts a region of interest
reshape(cn, rows)Changes shape without copying data
release()Releases the data (important for memory management)

Memory Management

In Java/OpenCV, you should call release() on Mat objects when done to free native memory. Unlike regular Java objects, Mat data is allocated in native memory and won't be freed by garbage collection alone.

Mat img = Imgcodecs.imread("image.jpg");
// ... use the image ...
img.release();  // Free native memory
