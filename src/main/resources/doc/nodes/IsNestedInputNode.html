Is-Nested Input Node

Allows a pipeline to be run both as a nested sub-pipeline inside a Container, and as a standalone root/primary pipeline.

Purpose

When building reusable processing pipelines, you often want to:

  • Test the pipeline standalone - using a File Source or Webcam as input
  • Embed it inside a Container - receiving frames from the parent pipeline

The Is-Nested Input node automatically selects the correct input based on how the pipeline is being executed.

Inputs

  • Input 1 (Nested) - Connect to Container Input. Used when this pipeline runs inside a Container, receiving frames from the parent pipeline.
  • Input 2 (Root) - Connect to a source like File Source or Webcam. Used when this pipeline runs as the primary/root pipeline.

Example Setup

Container Input ──┐
                  ├──► Is-Nested Input ──► Processing ──► ...
File Source ──────┘

When this pipeline runs inside a Container, it uses Container Input. When opened and run directly, it uses the File Source.

Tip: Use Is-Nested Input at the start of reusable sub-pipelines. Connect Container Input to Input 1 and a test source (File/Webcam) to Input 2. This lets you test the sub-pipeline standalone before embedding it.
Note: The routing decision is made automatically based on the isEmbedded flag set by the pipeline executor.
