Is-Nested Output Node

Allows a pipeline to work both as a nested sub-pipeline inside a Container, and as a standalone root/primary pipeline.

Purpose

When building reusable processing pipelines, you often want to:

  • Test the pipeline standalone - sending output to a Monitor for viewing
  • Embed it inside a Container - sending output back to the parent pipeline

The Is-Nested Output node automatically routes to the correct output based on how the pipeline is being executed.

Outputs

  • Output 1 (Nested) - Connect to Container Output. Used when this pipeline runs inside a Container, sending frames back to the parent pipeline.
  • Output 2 (Root) - Connect to a Monitor or other endpoint. Used when this pipeline runs as the primary/root pipeline for viewing/testing results.

Example Setup

                              ┌──► Container Output
... ──► Processing ──► Is-Nested Output
                              └──► Monitor (for standalone testing)
Tip: Use Is-Nested Output at the end of reusable sub-pipelines. Connect Output 1 to Container Output and Output 2 to a Monitor or other endpoint for standalone testing.
Note: When nested, only Output 1 receives frames. When running as root, only Output 2 receives frames. The inactive output receives nothing.
