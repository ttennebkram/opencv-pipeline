Container Node

A Container node encapsulates a sub-pipeline, allowing you to organize complex processing chains into reusable, modular components.

How It Works

A Container acts as a "black box" that contains its own internal pipeline. When a frame enters the Container:

  1. The frame is passed to the Container Input node inside
  2. The internal pipeline processes the frame
  3. The result from Container Output is returned

Editing a Container

Double-click a Container node to open its sub-pipeline editor window. Inside, you'll see:

  • Container Input - Receives frames from the parent pipeline
  • Container Output - Sends processed frames back to the parent
  • Your processing nodes connected between them

External Pipeline Files

Containers can reference external .json pipeline files, allowing you to:

  • Reuse the same pipeline in multiple places
  • Edit the sub-pipeline in its own window
  • Share pipeline components between projects

Set the pipeline file path in the Container's properties dialog.

Tip: Use Containers to break down complex pipelines into manageable pieces. Each Container can be tested independently.
Note: Changes to an external pipeline file are reflected wherever that Container is used.
