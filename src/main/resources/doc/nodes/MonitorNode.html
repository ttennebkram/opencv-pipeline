Monitor Node

A passthrough node used to observe frames at any point in the pipeline without modifying them.

Purpose

The Monitor node serves as an inspection point in your pipeline. Click on a Monitor node to view its output in the preview pane, allowing you to debug and verify processing at intermediate stages.

Behavior

  • Passes frames through unchanged (identity operation)
  • Creates a clone of each frame for the thumbnail/preview
  • Cannot be disabled (always passes through)
Tip: Insert Monitor nodes between processing stages to see how each step transforms the image. This is invaluable for debugging filter chains.

Use Cases

  • Debugging - see intermediate results in a pipeline
  • Verification - confirm a filter is working as expected
  • Documentation - mark important observation points
Note: Monitor nodes have minimal performance impact but do allocate memory for the frame clone used in thumbnails.
