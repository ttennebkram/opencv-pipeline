Divide By Background Node

Corrects for uneven illumination by estimating the background using a heavy blur and dividing the image by it.

Algorithm

background = GaussianBlur(input, large_kernel)
output = normalize(input / (background + epsilon))

Properties

  • Kernel Size - Size of blur kernel for background estimation (should be large)
  • Epsilon - Small value to prevent division by zero

How It Works

In images with uneven lighting (e.g., microscopy, scanned documents), dividing by the estimated background normalizes the brightness across the image. Dark regions become brighter and bright regions become normalized.

Applications

  • Microscopy image correction
  • Document scanning cleanup
  • Compensating for vignetting
  • Flat-field correction
Tip: Use a kernel size large enough that the blur represents the background illumination, not the actual image content. Typically 20-100+ pixels depending on image size.
Note: The output is normalized to use the full dynamic range. Very bright or dark regions in the original may clip.
