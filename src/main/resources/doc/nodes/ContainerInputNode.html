Sub-pipeline Boundary Input Node

The entry point for frames entering a Container's sub-pipeline. This node receives frames from the parent pipeline and makes them available inside the container.

Location

Sub-pipeline Boundary Input nodes only appear inside Container sub-pipelines. They are automatically created when you create a new Container and cannot be deleted.

Behavior

  • Receives frames from the Container node's input in the parent pipeline
  • Has one output that connects to your processing nodes
  • Acts as a passthrough - no modifications to the frame
Important: Only one Sub-pipeline Boundary Input node is allowed per container. Attempting to add a second will show an error.
Tip: Position the Sub-pipeline Boundary Input on the left side of your sub-pipeline canvas for clarity. Connect its output to the first processing node in your chain.
