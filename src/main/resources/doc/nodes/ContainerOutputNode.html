Sub-pipeline Boundary Output Node

The exit point for frames leaving a Container's sub-pipeline. This node sends processed frames back to the parent pipeline.

Location

Sub-pipeline Boundary Output nodes are only active when inside Container sub-pipelines. They are automatically created when you create a new Container and cannot be deleted.

Grayed Out? If this node appears grayed out, it means the pipeline is being viewed at the root level (not inside a container). Sub-pipeline Boundary nodes only function when running inside a Container - they have no effect when the pipeline runs standalone. This visual indicator helps you understand the execution context.

Behavior

  • Has one input that receives frames from your processing chain
  • Sends processed frames back to the parent pipeline
  • Acts as a passthrough - no modifications to the frame
Important: Only one Sub-pipeline Boundary Output node is allowed per container. Attempting to add a second will show an error.
Tip: Position the Sub-pipeline Boundary Output on the right side of your sub-pipeline canvas. Connect your final processing node's output to it.
