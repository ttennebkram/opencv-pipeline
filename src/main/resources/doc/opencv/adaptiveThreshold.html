Adaptive Threshold

Applies an adaptive threshold to an image. Unlike simple thresholding, the threshold value is calculated for each pixel based on a local neighborhood.

Signature

Imgproc / cv2.adaptiveThreshold(Mat src, Mat dst, double maxValue, int adaptiveMethod, int thresholdType, int blockSize, double C)

Parameters

  • Mat src - Source 8-bit single-channel image.
  • Mat dst - Destination image of the same size and type as src.
  • double maxValue - Non-zero value assigned to pixels that satisfy the condition.
  • int adaptiveMethod - Adaptive thresholding algorithm: ADAPTIVE_THRESH_MEAN_C or ADAPTIVE_THRESH_GAUSSIAN_C.
  • int thresholdType - Must be THRESH_BINARY or THRESH_BINARY_INV.
  • int blockSize - Size of pixel neighborhood used for threshold calculation. Must be odd (3, 5, 7, ...).
  • double C - Constant subtracted from the calculated mean or weighted mean.

Description

Adaptive thresholding is particularly useful when:

  • Lighting varies across the image
  • There are shadows or gradients
  • Global thresholding produces poor results

Parameter tuning:

  • blockSize: Larger values = smoother threshold transitions. Typical: 11-31.
  • C: Positive C = darker threshold (more white pixels). Negative C = brighter threshold. Typical: 2-10.

Common use cases:

  • Document/text binarization
  • Processing images with uneven lighting
  • Segmenting objects from variable backgrounds
