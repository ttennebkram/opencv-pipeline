CLAHE - Adaptive Histogram Equalization

Contrast Limited Adaptive Histogram Equalization. Improves contrast locally while preventing over-amplification of noise.

Signature

Imgproc / cv2.createCLAHE(double clipLimit, Size tileGridSize)

Parameters

  • double clipLimit - Threshold for contrast limiting. Higher values give more contrast. Default 40.0.
  • Size tileGridSize - Size of grid for histogram equalization. Default (8, 8).

Description

CLAHE divides the image into small tiles and applies histogram equalization to each tile. Unlike global histogram equalization, this:

  • Works well with varying lighting conditions
  • Prevents noise amplification through clip limiting
  • Produces more natural-looking results

How it works:

  1. Divide image into tiles (e.g., 8x8 grid)
  2. Compute histogram for each tile
  3. Clip histogram at clipLimit (redistribute excess)
  4. Apply histogram equalization to each tile
  5. Interpolate boundaries to remove tile artifacts

Example:

// Create CLAHE object
CLAHE clahe = Imgproc.createCLAHE(2.0, new Size(8, 8));

// Apply to grayscale image
Mat enhanced = new Mat();
clahe.apply(gray, enhanced);

// For color images, apply to L channel in LAB space
Mat lab = new Mat();
Imgproc.cvtColor(color, lab, Imgproc.COLOR_BGR2Lab);
List<Mat> channels = new ArrayList<>();
Core.split(lab, channels);
clahe.apply(channels.get(0), channels.get(0));
Core.merge(channels, lab);
Imgproc.cvtColor(lab, result, Imgproc.COLOR_Lab2BGR);

Typical parameter values:

  • clipLimit: 2.0-4.0 (lower = less contrast, higher = more)
  • tileGridSize: (8,8) to (16,16)
