goodFeaturesToTrack (Shi-Tomasi)

Finds the strongest corners in an image using the Shi-Tomasi method. Returns corner locations directly, ready for tracking.

Signature

Imgproc / cv2.goodFeaturesToTrack(Mat image, MatOfPoint corners, int maxCorners, double qualityLevel, double minDistance)

Parameters

  • Mat image - Input 8-bit or 32-bit single-channel image.
  • MatOfPoint corners - Output vector of detected corners.
  • int maxCorners - Maximum number of corners to return. 0 means no limit.
  • double qualityLevel - Minimum accepted quality (0-1). Corners with quality below maxQuality * qualityLevel are rejected.
  • double minDistance - Minimum Euclidean distance between returned corners.

Description

The Shi-Tomasi method modifies Harris corner detection by using the minimum eigenvalue instead of the Harris formula:

R = min(λ1, λ2)

This often produces better corners for tracking applications.

Advantages over Harris:

  • Returns corner locations directly (not a response map)
  • Built-in non-maximum suppression via minDistance
  • Quality thresholding is automatic
  • Better suited for optical flow and tracking

Example:

MatOfPoint corners = new MatOfPoint();
Imgproc.goodFeaturesToTrack(gray, corners, 100, 0.01, 10);

// Draw detected corners
Point[] pts = corners.toArray();
for (Point pt : pts) {
    Imgproc.circle(dst, pt, 5, new Scalar(0, 255, 0), -1);
}

Typical parameter values:

  • maxCorners = 100-500
  • qualityLevel = 0.01-0.1
  • minDistance = 10-30 pixels
